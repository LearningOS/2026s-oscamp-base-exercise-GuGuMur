1. 基础线程管理 (Basic Threading)
thread::spawn 与 move 闭包
这是 Rust 创建线程最常用的方式。

spawn: 创建一个新线程。新线程的生命周期可能比创建它的主线程长（除非主线程结束，进程随之退出）。

move 关键字: 由于 Rust 有严格的所有权机制，当你在子线程中使用主线程的变量时，必须使用 move 闭包将变量的所有权转移到子线程中。否则，Rust 无法保证变量在子线程执行期间依然有效。

JoinHandle
thread::spawn 会返回一个 JoinHandle。

阻塞等待: 调用 handle.join() 会阻塞当前线程，直到对应的子线程运行结束。

获取返回值: join() 返回一个 Result，如果子线程成功运行结束，它会包裹闭包的返回值。

2. 高级线程操作 (Advanced Operations)
线程配置：thread::Builder
如果你需要更精细地控制线程（如自定义线程名、设置栈大小），需要使用 Builder：

命名: builder.name("worker".into()) 在调试和性能分析时非常有用。

栈大小: 可以根据任务需求手动调整（默认通常为 2MB）。

线程局部存储：thread_local!
这是一种特殊的静态变量。

隔离性: 每个线程都会拥有一份该变量的独立拷贝。

安全性: 由于数据不跨线程共享，因此不需要复杂的锁（Mutex）即可实现内部可变性（通常配合 RefCell 使用）。

访问: 必须通过 .with(|val| { ... }) 方法进行闭包操作。

作用域线程：thread::scope (Rust 1.63+)
这是 Rust 并发编程的一个重大改进。

解决借用问题: 普通的 spawn 要求数据必须是 'static 的（即拥有所有权或长久有效）。而 scope 创建的线程保证在作用域结束前被 join，因此它们可以安全地借用栈上的数据（即引用 &T）。

性能: 避免了不必要的数据克隆（Clone）或原子引用计数（Arc）。

3. 并发安全基石：Send 与 Sync
这是 Rust 区别于其他语言的核心：

Send: 标记一个类型可以跨线程转移所有权。几乎所有类型都是 Send，但 Rc<T> 不是（因为它不是原子的，跨线程会导致引用计数出错）。

Sync: 标记一个类型可以在多个线程间安全地被引用（即 &T 是 Send）。

数据竞争预防: Rust 编译器在编译期检查这些特征。如果你尝试在一个线程中访问非 Sync 的数据且没有加锁，编译会报错。
